# 비밀번호 미설정 소셜 가입 계정의 이메일 로그인 처리

## 배경

### 시나리오

1. 사용자가 구글 로그인으로 가입
2. 온보딩 플로우를 거치지 않아 비밀번호가 설정되지 않은 상태
3. 이후 이메일 로그인 시도

### 문제점

- 비밀번호가 없어 **비밀번호 입력 인풋을 보여주는 것은 부적절**
- 이미 가입된 계정이므로 **새로 가입을 유도하는 것도 부적절**

## 제안

### 처리 방식: 소셜 로그인 유도

비밀번호가 설정되지 않은 소셜 가입 계정이 이메일 로그인을 시도할 경우, **소셜 로그인으로 유도**하는 것을 제안합니다.

### 이후 플로우

1. 사용자가 소셜 로그인 진행
2. 온보딩에서 비밀번호 설정
3. 이후에는 이메일 로그인도 가능

## 기획 의도

### 소셜 로그인 유도를 제안하는 이유

- 기존 온보딩 플로우를 그대로 활용
- "비밀번호가 설정되지 않았다"는 상태를 사용자에게 노출하지 않음
- 자연스러운 사용자 경험

### 개인정보보호 고려사항

- 어떤 소셜 계정(구글/애플)으로 가입했는지는 **알려주지 않음**
- "소셜 로그인으로 가입된 계정"이라고만 안내
- 이유: 제3자가 특정 이메일의 가입 방식을 파악하는 것을 방지
