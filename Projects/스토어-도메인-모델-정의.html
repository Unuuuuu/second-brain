목표

스토어 도메인에 존재하는 도메인 모델들을 명확하게 정의한다.

설계 원칙

  • 순수 도메인 모델: 각 모델은 자기 고유 속성만 정의
  • 관계는 조합으로 처리: 다른 모델과의 관계는 사용처에서 조합

새 모델 추가 시 확인

  • 다른 모델 배열을 직접 포함하지 않았는가?
  • 관계는 사용 예시로 보여주었는가?

장바구니 관련

장바구니 계층 구조

장바구니 API는 다음과 같은 4단계 계층 구조를 가집니다:

Cart (장바구니)
└── DeliveryGroup (배송 그룹) - 파트너/배송템플릿 기준
    └── OrderProduct (주문 상품) - 상품 1개
        └── OrderProductOption (주문 상품 옵션) - 장바구니에 담은 1건
            └── SetOption[] (세트 옵션) - 조합된 개별 옵션들

실제 예시: 티셔츠 구매

🛒 장바구니
└── 📦 배송그룹: "NCP 파트너"
    └── 👕 주문상품: "나이키 드라이핏 티셔츠"
        ├── 🏷️ 주문상품옵션 #1 (cartNo: 101)
        │   ├── optionTitle: "L / 블랙"
        │   ├── orderCnt: 2
        │   └── setOptions:
        │       ├── { optionName: "사이즈", optionValue: "L" }
        │       └── { optionName: "색상", optionValue: "블랙" }
        │
        └── 🏷️ 주문상품옵션 #2 (cartNo: 102)
            ├── optionTitle: "M / 화이트"
            ├── orderCnt: 1
            └── setOptions:
                ├── { optionName: "사이즈", optionValue: "M" }
                └── { optionName: "색상", optionValue: "화이트" }

각 레벨의 역할

레벨모델의미예시
1CartDeliveryGroup배송 그룹”NCP 파트너 배송”
2CartProduct상품 자체”나이키 드라이핏 티셔츠”
3CartOption장바구니 담기 1건”L/블랙 2개” ← cartNo 1개
4CartSetOption[]옵션 조합의 구성요소사이즈=L, 색상=블랙

옵션 선택 방식 (selectType)

  • FLAT (단독형): setOptions에 1개만 존재

  • COMBINATION (조합형): setOptions에 여러 개 존재
