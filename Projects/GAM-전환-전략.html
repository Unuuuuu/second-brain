Jan 06, 202616 min read

Adrop → Google Ad Manager 전환 전략

1. 요약

항목내용
현재Adrop 광고 플랫폼
목표Google Ad Manager (GAM) + 자체 광고주 어드민
예상 기간3-4개월
예상 비용개발 3,000-5,000만원 / 운영 월 50-100만원

2. 왜 Google Ad Manager인가?

2.1 검토한 대안들

서비스직접 광고광고주 어드민결론
Google Ad Manager✅❌ (자체 구현)채택
AppLovin MAX❌❌탈락
완전 자체 구축✅✅비용 과다 (6개월+, 2억+)

AppLovin MAX: Direct Sold 기능이 2025년 초 제거되어 직접 광고 불가

2.2 GAM 선택 이유

  1. 직접 광고 캠페인 완벽 지원 - 광고주와 직접 계약 가능
  2. 무료 사용 - 한국 기준 월 2억 노출까지 무료
  3. 이미지 호스팅 무료 - Google CDN에서 서빙
  4. 높은 자유도 - 커스텀 사이즈, 다양한 레이아웃 지원
  5. 글로벌 신뢰성 - Google 인프라 활용

3. GAM 가격 정책

3.1 무료 한도 (한국 기준)

광고 유형월간 무료 한도
일반 광고 (배너, Native 등)2억 노출
비디오 광고80만 노출

3.2 초과 시 비용

항목비용
초과 시약 17원 / 1,000노출
예: 1억 노출 초과 시약 170만원/월

3.3 군돌이 현재 사용량

항목수치
월간 총 노출약 3,000만 노출
무료 한도 대비15% (충분히 여유)

6. 과금 방식

GAM은 다양한 과금 방식을 지원합니다.

과금 방식설명용도
CPM1,000 노출당 비용일반 캠페인
CPC클릭당 비용성과 기반 광고
CPA전환당 비용 (구매, 가입 등)전환 최적화
CPD일당 비용기간 보장 광고
CPP (기간제)주/월 단위 비용CPD를 기간으로 환산하여 적용

7. 소재 승인 워크플로우

광고주가 업로드한 소재를 검토하고 승인/반려하는 프로세스입니다.

7.1 워크플로우

단계설명
1. 소재 제출광고주가 이미지, 텍스트 업로드
2. 검토 대기관리자에게 알림 발송
3. 관리자 검토승인 / 반려 / 수정 요청 선택
4. 결과 통보광고주에게 결과 및 사유 전달
5. 광고 시작승인된 소재만 GAM에 업로드되어 송출

7.2 소재 상태

상태설명
검토 대기제출 후 관리자 검토 대기 중
승인됨검토 통과, 광고 송출 시작
반려됨검토 불통과, 수정 후 재제출 필요
수정 요청일부 수정 필요, 코멘트 확인 후 재제출
송출 중현재 앱에서 노출 중

7.3 알림

이벤트수신자알림 방식
소재 제출관리자이메일, 슬랙
승인 완료광고주이메일
반려광고주이메일
수정 요청광고주이메일

8. 팝업 광고 (캐러셀) 지원

8.1 현재 기능

현재 goondori 앱의 메인 팝업 광고는 다음 기능을 제공합니다:

  • 캐러셀 형태: 여러 광고를 스와이프로 탐색
  • 오늘 다시 보지 않기: 사용자가 원하면 하루 동안 미노출
  • 개별 광고 닫기: 각 광고별로 닫기 가능

8.2 GAM 지원 여부

기능GAM 기본 지원자체 구현 필요
여러 광고 로테이션✅-
캐러셀 UI❌✅
“오늘 다시 보지 않기”△ (시간 기반만)✅
각 광고별 개별 닫기❌✅

GAM은 한 번에 1개 광고만 제공하므로, 캐러셀처럼 여러 개를 동시에 보여주는 기능은 자체 구현 필요

8.3 해결 방안

광고 유형별 처리 방식:

광고 유형처리 방식
배너 광고GAM 사용
네이티브 광고GAM 사용
팝업/캐러셀 광고자체 시스템 구현

팝업 광고는 자체 시스템으로 구현하여:

  • 캐러셀 UI 자유롭게 구성
  • “오늘 다시 보지 않기” 등 커스텀 로직 적용
  • 광고별 닫기 옵션 설정 (없음 / 1회 / 오늘 / 일주일 / 영구)

9. 리포트 및 성과 분석

9.1 GAM 제공 리포트 항목

GAM은 배너/네이티브 광고에 대해 자동으로 상세 리포트를 제공합니다:

카테고리항목
기본 지표노출 수, 클릭 수, 클릭률 (CTR)
수익총 수익, eCPM, 과금 방식별 수익
뷰어빌리티실제 화면에 보인 비율
분석 축일자별, 광고별, 광고주별, 지역별, 기기별

9.2 통합 리포트 (GAM + 팝업)

팝업 광고는 자체 구현이므로 GAM 리포트에 자동 포함되지 않습니다. 해결 방안: 통합 리포트 대시보드 구축

데이터 소스조회 방식포함 항목
GAMGAM API로 조회배너, 네이티브 노출/클릭/수익
자체 DB직접 조회팝업 노출/클릭/수익

광고주가 보는 통합 리포트 예시:

광고 유형노출클릭CTR수익
배너150,0001,5001.0%₩500,000
네이티브80,0008001.0%₩300,000
팝업50,0002,5005.0%₩200,000
합계280,0004,8001.7%₩1,000,000

광고주는 하나의 대시보드에서 모든 광고 유형의 성과를 통합하여 확인 가능

13. 커뮤니티 반응 및 실제 사용 경험

13.1 Direct Sold Campaigns

긍정적 반응:

  • 58% 퍼블리셔가 direct-sold에서 41%+ 수익 기대 (Digiday 조사)
  • 84% 광고 구매자가 direct deal에 예산 투입 의향
  • AppLovin MAX Direct Sold 기능이 2025년 초 제거되어 GAM이 사실상 유일한 대안

13.2 react-native-google-mobile-ads SDK

최신 이슈 (GitHub, 2025.12 기준):

이슈내용상태
#825NativeAssetView 태그 못 찾음Open
#824iOS NativeAdView 클릭 정책 위반Open
#826iOS 물리 기기 전면광고 닫기 불가Open

  • Google이 React Native 공식 지원 안 함 (Android, iOS, Unity, Flutter만 공식)
  • Expo 52+ / RN 0.76+ 사용 권장
  • FLUID 사이즈: Android에서 높이 계산 버그 있음, iOS는 정상

13.3 GAM API

현재 상태 (2024-2025):

  • 2024년 9월 Open Beta 출시
  • 일시 제거된 서비스: LineItem, Creative, Contact, Label, Team (데이터 불일치로)
  • 마이그레이션 데드라인: CustomTargetingValue (2025.07), Report dimensions (2025.10)

개발자 포럼 반응:

  • 랜덤 404 에러 간헐적 발생 보고
  • SOAP API 성능 이슈 보고
  • 학습 곡선 높음 (Order → LineItem → Creative → LICA 관계 이해 필요)

인증 설정:

  • OAuth2 Service Account 방식 권장
  • JSON 키 파일 + 네트워크 코드 필요
  • 클라이언트 라이브러리: Java, .NET, Python, PHP, Ruby 제공

13.4 GAM API 접근성

무료 버전(Small Business)에서도 API 접근 가능:

  • Admin → Global Settings → Network Settings에서 API 활성화
  • Order, LineItem, Creative 생성/수정 가능
  • 어드민 구축 가능 (공식 문서에 use case로 명시)

GAM 360에서만 가능한 것:

  • Open Bidding (서버사이드 헤더비딩)
  • Audience Management (고급 오디언스 세그먼테이션)
  • Google 직접 Support
  • 더 깊은 Ad Unit 계층 (5 levels vs 2 levels)
  • Data Studio/DMP 통합
