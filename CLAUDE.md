# Second Brain - Claude 컨텍스트

이 볼트는 PARA 방식으로 구조화된 개인 지식 관리 시스템입니다.

## 볼트 소유자

- 프론트엔드 개발자 (React Native, Expo)
- 관심 영역: 개발, 프리다이빙, 여행, 신앙, 크로스핏, 일상

---

## 폴더 구조

```
second-brain/
├── Daily/           # 데일리 노트 (스탠드업)
├── Projects/        # 마감일이 있는 작업
├── Areas/           # 지속적인 삶의 영역
├── Resources/       # 참고 자료
├── Archives/        # 완료/비활성 항목
└── Templates/       # 템플릿
```

---

## PARA 분류 가이드

### 분류 결정 흐름

```
1. "마감일/완료 시점이 있는가?"
   → Yes: Projects/
   → No: 다음 질문

2. "삶에서 지속적으로 관리해야 하는 영역인가?"
   → Yes: Areas/[해당영역]/
   → No: 다음 질문

3. "나중에 참고할 정보인가?"
   → Yes: Resources/
   → No: Archives/
```

### 폴더별 상세 설명

| 폴더 | 용도 | 예시 |
|------|------|------|
| **Projects** | 마감일이 있는 작업 | 앱 출시, 여행 계획, 자격증 취득 |
| **Areas** | 지속적으로 관리하는 삶의 영역 | 개발 학습, 프리다이빙, 신앙 |
| **Resources** | 재사용 가능한 참고 정보 | 맛집 리스트, 체크리스트, 레시피 |
| **Archives** | 완료되거나 비활성화된 항목 | 완료된 프로젝트, 과거 기록 |

### 분류 예시

| 기록 내용 | 분류 | 이유 |
|----------|------|------|
| "이번 달 앱 출시 준비" | `Projects/앱-출시-1월.md` | 마감일 있는 작업 |
| "React Native 성능 최적화 팁" | `Areas/Development/` | 개발 영역 지속 관리 |
| "프리다이빙 호흡법 정리" | `Areas/Freediving/` | 프리다이빙 영역 |
| "2월 일본 여행 계획" | `Projects/일본-여행-2월.md` | 마감일 있는 계획 |
| "크로스핏 WOD 기록" | `Areas/Fitness/` | 운동 영역 |
| "오늘 묵상 내용" | `Areas/Faith/` | 신앙 영역 |
| "맛집 리스트" | `Resources/맛집-리스트.md` | 참고 정보 |
| "여행 패킹 체크리스트" | `Resources/여행-패킹-체크리스트.md` | 재사용 참고 정보 |

### Areas 하위 폴더 (필요시 생성)

- `Areas/Development/` - 개발 관련
- `Areas/Freediving/` - 프리다이빙
- `Areas/Fitness/` - 크로스핏, 운동
- `Areas/Faith/` - 신앙생활
- `Areas/Travel/` - 여행 (지속적 관심사)
- `Areas/Life/` - 일상적인 생각, 기타

## Claude 행동 규칙

### 노트 생성 시

1. **분류 판단**: 위 분류 흐름에 따라 적절한 폴더 결정
2. **바로 저장**: 애매한 경우에도 Claude가 판단해서 저장
3. **이유 설명**: 저장 시 왜 해당 폴더인지 간단히 설명
4. **폴더 생성**: 필요한 하위 폴더가 없으면 생성

### 프로젝트 완료 시

- 완료된 프로젝트는 `Archives/`로 이동 제안
- 프로젝트에서 추출한 지식은 `Areas/` 또는 `Resources/`로 분리 가능

---

## 노트 작성 규칙

### 파일 네이밍

- **데일리 노트**: `YYYY-MM-DD.md` (예: 2026-01-02.md)
- **일반 노트**: 한글 명사형, 하이픈 구분 (예: `크로스핏-루틴.md`)

### 링크 형식 (GitHub 호환)

상대 경로 마크다운 링크 사용:

```markdown
[홈](./Home.md)
[크로스핏 루틴](./Areas/Fitness/크로스핏-루틴.md)
[오늘 데일리](./Daily/2026-01-02.md)
```

### Frontmatter

```yaml
---
date: 2026-01-02
---
```

## 핵심 원칙

1. **단순함 유지**: 과하게 구조화하지 않음
2. **필요시 확장**: 빈 폴더 미리 만들지 않음
3. **기록 우선**: 분류보다 기록이 중요, 분류는 나중에 정리 가능
4. **Home.md 중심**: 시작점에서 주요 영역으로 연결
